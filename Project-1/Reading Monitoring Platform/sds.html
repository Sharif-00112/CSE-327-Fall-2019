    3. Description of Design Components

    3.1 Components Overview

    We will be giving a basic overview of the various components to be used in our system.

    3.2 Database

    SqLite will be used as our database software for Hand held software application and PostgreSQL will be used for our web applications. To represent various data from our project, we will be using tables included within the database software.

    3.3 Server Software

    Through our server software, we will be able to directly manipulate the contents of our database system based on commands provided from the server procedure proxy.

    3.4 Server procedure proxy

    we will be provided an interface for us to request information or submit updates to the server through the server procedure proxy. The clients (Teacher) will be given limited access to the server so that they can update the contents._

    3.5 Client Software

    The client software on which we will present our project will either be on a desktop computer or a hand held device. The software will have two general purposes so that the users can easily update information on the project. Our users will mainly consist of two variants, (Teacher and Student). So for this the software will mainly consist of two interface to accomodate the two different variants.

    • Hand held Client Software
      The hand held client software will allow both type of users to view their varient of the software and they can also make changes and updates relevent to the inteface assign to them.

    • Desktop Client Software
      The Desktop client software will allow both type of users to view their varient of the software and they can also make changes and updates relevent to the inteface assign to them.

    5.1 Class Name: Registration class

    Purpose: Students and teacher both are responsible for maintaining an aacurate registration
    record. Students are to arrange to meet with their academic advisor through registration class. In
    registration class there are some attributes such as fullName,email,username,password,conPass. This
    class also have some methods.

     5.1.1 Attribute

    1. Name: fullName
      Type: string
      Purpose: It decleared the user's full name.

    2. Name: email
      Type: string
      Purpose: It decleared the user's email.

    3. Name: username
      Type: string
      Purpose: It decleared the user's name.

    4. Name: password
      Type: string
      Purpose: It decleared the user's password.

    5. Name: conPass
      Type: string
      Purpose: It decleared the confirmation of user's password.

     5.1.2 Methods

    1. Name: username
      Parameters:(username, string and it calls the username)
      Return type: string
      Purpose and description: It decleared the username. In every registration every person has specific user name.
    2. Name: password
      Parameters:(password, string and it calls the user's password)
      Return type: string
      Purpose and description: It decleared user's password. Every user has different password.
    3. Name: conPass
      Parameters:(conPass, string and it calls confirm password)
      Return type: string
      Purpose and description: It decleared the confirmation of password. When a user can give their password then there are another options fpr confirmation password.
    4. Name: email
      Parameters:(email, string and it calls the user email)
      Return type: string
      Purpose and description: It decleared user's email. Every user has specific email address for their own. They can use their email address in the registration form.
